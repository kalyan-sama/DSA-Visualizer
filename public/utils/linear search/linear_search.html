Linear Search Algorithm

Basic Definition

Linear search, also known as sequential search, is a simple searching algorithm that checks each element in a list sequentially until a match is found or the entire list has been searched.

How It Works

  1. Start from the first element of the list.
  2. Compare the current element with the target value.
  3. If they match, return the current position.
  4. If they don't match, move to the next element.
  5. Repeat steps 2-4 until the element is found or the end of the list is reached.
  6. If the end of the list is reached without finding the element, return a signal (e.g., -1) to indicate the element was not found.

Time and Space Complexity

  • Time Complexity:
    • Best Case: O(1) - when the target element is at the beginning
    • Worst Case: O(n) - when the target element is at the end or not present
    • Average Case: O(n) - on average, need to search half the list
  • Space Complexity: O(1) - only requires a constant amount of additional space

Advantages

  • Simple to understand and implement
  • Works on unsorted lists
  • No need for additional data structures
  • Efficient for small lists
  • Can be used on lists of any data type

Disadvantages

  • Inefficient for large lists
  • Slower compared to other search algorithms like binary search for sorted lists
  • Not suitable for large datasets in time-critical applications

Use Cases

  • Searching in small unsorted lists
  • Searching for a known position (e.g., first or last occurrence)
  • One-time searches where setting up complex data structures isn't worth it
  • Searching in data structures that don't support random access (e.g., linked lists)
  • As a subroutine in more complex algorithms
  • In situations where simplicity is more important than efficiency
